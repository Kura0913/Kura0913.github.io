Unreal Engine設定及除錯

發表於 2023-07-20 更新於 2024-03-20 分類於 Unreal Engine ， Unreal Engine 設定相關

前言

這些都是我在開發的過程中，有遇到需要調整的項目，之後如果有遇到新的問題，也會持續更新在這篇文章

Visual Studio版本更改

有些人使用的Visual Studio版本或許不是2022的版本，或是使用其他的編譯器，但又不想重新下載，其實是有解決辦法的，點擊左上角
Edit->Editor Preferences->General->Source code， 會看到”Source Code Editor”，點開來改成自己想使用的編譯器就行了

解決切換畫面後，畫面卡頓問題

在模擬的時候常常會畫面切來切去，但只要一點到Unreal以外的視窗，模擬的畫面就會非常卡頓，這時可以透過更改設定來解決，
Edit->Editor Preferences->General->Performance，把”Use Less CPU when in Background”取消掉，就可以解決這個問題

打包地圖缺失

打包地圖的時候，可能會遇到地圖只打包了你正在使用的地圖，或是預設地圖，此時就要透過設定來修改打包的目標，建議每次的打包，都要執行接下來的設定
Edit->Project Settings->Project->Maps & Modes，下方有個”Default Maps”的設定，建議將Editor Startup Map和Game Default Map都換成執行檔開啟執行時，第一張要執行的地圖

接著，到Project->Packaging->Packaging->Advanced，有一個”List of maps to include in a packaged build”的選項，點擊”加號”之後，到專案的資料夾下找出想打包的地圖，有幾張就加幾張

這樣打包時就會把在List裡的地圖通通打包進去

開啟檔案失敗

在執行C++專案時，如果專案是有換過電腦執行，可能會出現以下錯誤:

上面會顯示在某個路徑上無法找到”YourProjectName.uproject”的檔案

正常來說，在移動完專案之後， 會對”YourProjectName.uproject”點擊右鍵，選擇”Generate Visual Studio project files”重新生成專案文件，在這個過程中檔案相關的路徑也會被更新，但是有時候啟動檔的路徑不會被更新到，這時候就要手動調整

啟動檔的路徑儲存在 .\YourProjectFolder\Intermediate\ProjectFiles\YourProjectName.vcxproj.user

把這個檔案打開會有下列內容，其中紅色框框處就是啟動檔的路徑

這邊有兩個選擇：

  • 1、改成現在電腦上的路徑

  • 2、改成相對路徑

我個人推薦第2種方法，可以避免之後專案移動時又發生一樣的問題

修改方式就是將紅色框框處的路徑改為：($SolutionDir)YourProjectName.uproject

修改後儲存，專案就可以順利啟動
